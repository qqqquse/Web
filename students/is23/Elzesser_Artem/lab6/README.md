# Отчет по ЛР 6: Backend на Node.js

## Цель: Создание серверной части приложения на Node.js с архитектурой MVC и интеграцией с клиентским приложением.
### Теоретическая часть
1. Node.js - серверный JavaScript

    Назначение: Выполнение JS-кода вне браузера

    Основа: Движок V8 от Google Chrome

* Возможности:

    * Веб-серверы (BFF - Backend For Frontend)

    *  CLI-утилиты и скрипты автоматизации

    *  Десктопные и мобильные приложения

    *  Микроконтроллеры

2. Архитектура серверного приложения

* Слоистая архитектура (MVC-подобная):

  *  Controller - обработка HTTP-запросов/ответов

  *  Service - бизнес-логика приложения

   * Repository/DAO - работа с данными

   * Database - хранение данных

 ### Практическая часть

Этапы разработки:
1. "Голый" HTTP-сервер (нативный)

  *  Использование встроенного модуля http

  *  Простая обработка запросов

   * Ограниченная функциональность

2. Express.js - веб-фреймворк

* Упрощение работы с маршрутизацией

* Middleware для промежуточной обработки

* Поддержка REST API

3. REST API для акций (Stocks)

  *  GET /stocks - получение всех акций

  *  GET /stocks/:id - получение конкретной акции

  *  POST /stocks - создание новой акции

  *  DELETE /stocks/:id - удаление акции

4. Архитектурные паттерны

  *  DAO (Data Access Object) - абстракция доступа к данным

   * Repository - работа с хранилищем данных

   * Service - бизнес-логика

   * Controller - обработка HTTP



### Ключевые компоненты
1. DBConnector

   * Чтение/запись JSON-файлов

   * Абстракция над файловой системой

2. StocksRepository

   * Низкоуровневый доступ к данным

   * CRUD операции с файлом

3. StockDAO

   * Высокоуровневый доступ к данным

   * Валидация данных

   * Преобразование форматов

   * Паттерн "Синглтон"

4. StocksService

   * Бизнес-логика приложения

   * Обработка данных перед отдачей

5. StocksController

   * Обработка HTTP-запросов

   * Валидация входных данных

   * Формирование ответов

   * Обработка ошибок

### Интеграция с клиентом
Замена клиентского кода:

* Удаление статических данных getData()

* Реализация fetch запросов к серверу

* Обработка асинхронных операций

* Обработка ошибок сети

Пример клиентского запроса:

```
// Вместо:
const data = getData();

// Теперь:
const response = await fetch('http://localhost:8000/stocks');
const data = await response.json();
```

### Преимущества подхода
1. Разделение ответственности

  *  Клиент → отображение данных

  * Сервер → логика и хранение данных

2. Масштабируемость

  *  Легкое добавление новых endpoint'ов

   * Простое изменение бизнес-логики

  *  Возможность замены хранилища данных

3. Безопасность

* Валидация данных на сервере

* Контроль доступа к данным

* Защита от инъекций

4. Переиспользуемость

* Один API для разных клиентов

* Мобильные приложения

*  Другие веб-приложения
