# Отчет по ЛР 5: Веб-приложение с Fetch API

## Цель: Изучение современных методов работы с API через Fetch API, Promise и async/await.
 ### Теоретическая часть
1. Promise (Обещания)

    Назначение: Обработка асинхронных операций

    Состояния: Pending (выполняется), Fulfilled (успешно), Rejected (ошибка)

    Методы: .then(), .catch(), .finally()

2. Async/Await

    Синтаксический сахар над промисами

    Упрощает чтение асинхронного кода

    Использует try/catch для обработки ошибок

3. Fetch API

    Современная замена XMLHttpRequest

    Возвращает Promise

    Поддерживает различные методы HTTP

    Встроенная обработка JSON

### Практическая часть

Сравнение подходов:

XMLHttpRequest (предыдущая работа):

   * Использование колбэков

  *  Сложная обработка ошибок

   * Ручной парсинг JSON

Fetch API (текущая работа):

   * Работа с промисами

   * Простая обработка через try/catch

  *  Автоматический парсинг JSON методом .json()

 
 Преимущества Fetch API

  *  Чистый код - меньше вложенности и колбэков

  *  Современный стандарт - поддержка всеми браузерами

  *  Удобная обработка - цепочки промисов и async/await

  *  Гибкость - настраиваемые параметры запросов

   * Безопасность - встроенная CORS обработка

 Принципы работы

### Асинхронные операции:

  *  Запросы к серверу занимают время

  *  Блокировка кода недопустима

   * Promise позволяет продолжить выполнение других задач

### Цепочки вызовов:

  *  Последовательная обработка данных

  *  Обработка ошибок на каждом этапе

  *  Финализация операций через finally

### Обработка ответов:

  *  Проверка статуса ответа

  *  Преобразование форматов (JSON, текст, blob)

   * Обработка сетевых ошибок
